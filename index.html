📖 使用說明

一、基本操作

  • 新增持股：輸入股票代號、數量、單位（張/股），可選填買入均價以計算總損益。
  • 編輯持股：點擊列表中的「✎」按鈕，可修改股票資訊。
  • 刪除持股：點擊列表中的「✕」按鈕。
  • 分頁管理：支援 1~5 個自訂分頁，可在「進階設定」中調整數量。雙擊分頁名稱可重新命名。
  • 主題切換：點擊右上角圖示切換多種介面主題。

二、API 設定與限制

本軟體支援 Fugle API 以獲取即時股價。您可以在上方「進階設定」中輸入 API Key。

三、一組 Key 與兩組 Key 的差異

為了避免超過 API 速率限制而被禁用，系統會根據您輸入的 Key 數量動態調整更新頻率：

  • 一組 API Key：
    • 正常情況：每 20 秒更新一次。
    • 發生錯誤時：降至每 30 秒更新一次。
  • 兩組 API Key（主要 + 備用）：
    • 正常情況：每 10 秒更新一次，請求會在兩組 Key 間輪詢。
    • 發生錯誤時：降至每 15 秒更新一次。
  • 收盤時間：
    • 系統偵測到收盤（13:30 後或週末）時，自動調整為每 60 秒更新一次，以節省 API 額度。

💡 建議：若需追蹤較多股票（超過 10 支），建議申請兩組 Key 以獲得更快的更新頻率並分散風險。

四、資料備份

您可以在進階設定中將庫存與歷史記錄匯出為 JSON 檔案，或從備份檔還原。

五、國際指數與 Google 報價說明

除了台股加權指數外，其他國際股市指數（如道瓊、那斯達克等）是透過 Google 試算表 的公開資料抓取。

  • 為什麼不用 Fugle API？ 因為 Fugle 的方案主要針對台股資料，若要取得國際指數需要相當昂貴的企業方案。使用 Google 試算表作為免費的替代方案。
  • 為什麼不是即時的？ Google 試算表本身的資料就有大約 15 分鐘的延遲。此外，Google 伺服器會對公開發布的網頁進行快取（Cache）。
  • 更新頻率： 本系統設定為每 30 秒向 Google 發送一次請求，但實際上 Google 回傳的數據可能與真實市場有 15 分鐘以上的落差。這屬於免費資料源的先天限制。
